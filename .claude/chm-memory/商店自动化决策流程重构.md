本轮重构将商店自动化（shop_automation.gd）的决策流程从两个分支（自动买 + 手动检）统一为**一个循环**。

关键设计变更：

1. **停止条件收拢**：循环的停止条件只剩一种——决策结果为 `manual`（保留给玩家手动操作）。其他所有情况（skip、buy 等）继续循环。
2. **全买不起 / 客观不能**：这两种情况不再作为独立状态，而是归入"能否刷新"检查，由 `can_refresh()` 统一判定。
3. **不能刷新时直读作弊表**：当无法通过刷新获得新物品时（如钱用完且无刷新券），直接读取作弊表（cheat sheet）作为兜底决策，不再走完整的逻辑判断链。

原因：原始代码在两个分支间重复了大量逻辑，且停止条件分散（有些在 decider 层、有些在 automation 层），导致流程难以理解和调试。

如何运用：未来修改商店自动化逻辑时，所有停止判断集中在循环顶部检查 `manual` 一个条件；是否还能继续尝试均在 `can_refresh()` 中判定；兜底策略统一从作弊表直读。
